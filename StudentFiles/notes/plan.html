Class format

This unit is divided into three blocks:

  1. Introduction to programming games.
  2. Building games from components.
  3. The game development cycle.

Block 1: Introduction to programming games

In this block, each week:

  • we'll look at a new programming technique;
  • we'll see how to apply it to a game program; and,
  • you'll be given an exercise to test your understanding.

Block 2: Building games from components

In this block, each week:

  • we'll review an example game program; and,
  • you'll begin to develop your game program for your assessment.

Block 3: The game development cycle

In the final block, each week:

  • we'll examine the program development cycles; and,
  • you'll continue the development of your game program.

Dyslexia and learning difficulties

  • Make sure you have contacted Disability & Dyslexia Support (DDS).
  • Programming is particularly hard on your eyes.
  • The programs we're using in this unit are customisable for font and background colour.
  • Whether you have dyslexia or not, configuring your system can reduce eye strain.
  • The font we'll use in programming is Microsoft Consolas.
  • If you prefer to change the font, ensure that it clearly distinguishes the following characters:
    • 1 I l |
    • 0 O
    • ' "
    • : ;
    • . ,
    • () {} [] <>

Source control

  • Source control is a database for your program files.
  • All of your exercises and game programs should be developed and submitted through our source control system.
  • The source control system we are using in this unit is called subversion.
  • Our server is hosted by Google Code.
  • Your introductory class will cover this system and how to use it.

Exercises

  • In block 1, you are expected to complete one exercise a week.
  • The purpose of each exercise is to demonstrate your understanding of the topic covered in the class.
  • Exercises should not take long to complete: if you're quick you can complete them during the class; if you get stuck for more than a few minutes on one of the exercises, ask for help.

Game program

In blocks 2 and 3:

  • You will create a small game program.
  • Your program should demonstrate your understanding of the programming methods covered in class.
  • As with the exercises, this should not consume a significant portion of your time and may be completed during class time.

Assessment

Your exercises and game programs will be marked on:

  • how well you demonstrate your understanding of the topics covered in class; and,
  • consistent application of the covered programming principles.
