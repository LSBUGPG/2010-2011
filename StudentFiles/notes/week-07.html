My plan for this class:

  1. recap
  2. astyle
    • a program to format code for you
    • an easy way to increase your marks
    • an aid to debugging syntax problems
    • usage:
      • open a command prompt
      • cd to your source directory
      • eg. cd \Users\Paul\Documents\programming\sinnett
      • astyle will already be in your PATH if you have set up for SDL dlls
      • type: astyle --style=stroustrup *.cpp *.h
      • add -r to re-curse through folders
      • add -n to suppress backup files
    • also match settings in Visual Studio
  3. ping-pong
    • play through
    • based on "objects" project
    • walk-through from main
    • new programming principle "tell don't ask" (law of Demeter)
    • note console output to record scoring
  4. break
  5. no exercise this week
    • chance to catch up on previous exercises
    • start thinking about the game you want to make
    • choose participants for a focus group
    • demonstrate your understanding of:
      • abstraction
      • variables and functions
      • libraries
      • objects
    • you will get points for:
      • consistent style (use astyle to help)
      • sensible variable names
      • respecting the law of Demeter
      • submitting only the required files
      • useful comments (the comment must add something)
    • you will lose points for:
      • badly formatted code or inconsistent style
      • contradictory or bizarre variable names
      • missing files
      • warnings or errors in your build
      • contradictory or redundant comments
