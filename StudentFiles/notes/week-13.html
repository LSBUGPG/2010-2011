My plan for this class:

  1. tutorial: combining projects with the shell example
    • add in all the files from shell to your game
    • you will need to merge in code from common files
      • engine.cpp and engine.h
      • controls.cpp and controls.h
      • main.cpp
    • copy over missing functions and function prototypes
    • check for missing headers and libraries
    • check for missing initialisation and shut down code
    • empty out the code from game.cpp
    • replace with your object code
    • watch out for name collisions (eg. state)
    • signal the switch to the gameover state
    • replace all the placeholder images and text
  2. lecture: portfolios
    • home projects are the only thing that impresses programmers
    • it must be your own work, you will be quizzed on it
    • some programming jobs require programming tests
    • specific to game programming jobs
  3. reminder for your projects:
    • demonstrate your understanding of:
      • abstraction
      • variables and functions
      • libraries
      • objects
    • you will get points for:
      • consistent style (use astyle to help)
      • sensible variable names
      • respecting the law of Demeter
      • submitting only the required files
      • useful comments (the comment must add something)
    • you will lose points for:
      • badly formatted code or inconsistent style
      • contradictory or bizarre variable names
      • missing files
      • warnings or errors in your build
      • contradictory or redundant comments
  4. redistributing your game
    • build in release
    • make a folder for your game and include:
      • release exe file
      • any dll files you are using
      • any resource files (bitmaps audio)
      • any license files
    • test by corrupting your path to lib
    • optional: set fullscreen with SDL_FULLSCREEN
