Multi-platform programming

My plan for this class:

  1. lecture: multi-platform programming
    • a look at some other languages:
      • hello-world.c the original C language
      • hello-world.cpp C++ version using the stream library
      • hello-world.cs C# version
      • hello-world.m Objective C version
      • hello-world.java Java version
      • hello-world.js Javascript version
      • hello-world.lua Lua version
    • interpreted and compiled languages
      • directly interpreted
      • virtual machine
      • JIT compilation
      • native binary
    • parallel programming
      • programming with multiple threads
      • programming with multiple CPUs
      • atomic operations
      • deadlock
      • race condition
    • declarative programming vs imperative
      • "what is" rather than "how to"
      • programming without side-effects
      • useful in parallel programming
    • programming for Unity 3D
      • Javascript
      • each game object in Unity can run a script
      • scripts generally define an "Update" function
      • scripts can also declare instance specific data
      • signalling works through functions
      • similar to the objects and instances in our C++ programs
    • programming for iPhone or other platforms
      • might require a specific language (eg. Objective C for iPhone)
      • code and data must not rely on things that vary (eg. screen resolution)
      • you may need to build your programs into an existing framework (eg. Facebook)
  2. tutorial: project work
